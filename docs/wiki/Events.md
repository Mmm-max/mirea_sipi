# Управление событиями

Личные события — это задачи, встречи, блокировки времени и импортированные записи, которые отображаются в вашем персональном календаре и учитываются при поиске свободных слотов.

## Создание события

POST `/events` с телом:
```json
{
  "title": "Ревью кода",
  "description": "Ревью PR #42",
  "start_at": "2025-06-10T10:00:00Z",
  "end_at": "2025-06-10T11:00:00Z",
  "priority": "medium",
  "is_reschedulable": true,
  "visibility_hint": "private"
}
```

**Приоритеты событий:**
| Приоритет | Влияние на планирование |
|---|---|
| `low` | Мягкий конфликт, -10 баллов; если `is_reschedulable: true` — предлагается перенос |
| `medium` | Мягкий конфликт, -25 баллов |
| `high` | Мягкий конфликт, -45 баллов |
| `critical` | Жёсткий конфликт, слот недоступен полностью |

**Видимость (`visibility_hint`):**
- `public` (по умолчанию) — название видно другим участникам встречи
- `private` — другие видят только «Занят»

## Просмотр событий

- **Список всех событий:** GET `/events`
- **Одно событие:** GET `/events/:id`

## Редактирование события

PATCH `/events/:id` — передайте только те поля, которые хотите изменить:
```json
{
  "title": "Новое название",
  "priority": "high"
}
```

## Удаление события

DELETE `/events/:id` — событие удаляется немедленно.

## Импорт из календаря (.ics)

Если у вас уже есть события в другом календаре (Google Calendar, Outlook и т.д.):

1. Экспортируйте файл `.ics` из вашего приложения.
2. POST `/calendar-import/ics` — загрузите файл как `multipart/form-data`.
3. Статус импорта можно проверить через GET `/calendar-import/history`.

Повторный импорт того же файла обновляет существующие события, не создавая дубликатов.
