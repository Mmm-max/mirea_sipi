# Начало работы

## Регистрация

1. Отправьте POST-запрос на `/auth/register` с телом:
   ```json
   {
     "email": "user@example.com",
     "password": "ваш_пароль"
   }
   ```
2. В ответ вы получите пару токенов:
   - `access_token` — используется в заголовке `Authorization: Bearer <token>` для всех последующих запросов (действителен 15 минут)
   - `refresh_token` — используется для обновления access_token (действителен 30 дней)

> Email автоматически приводится к нижнему регистру. Пароль хранится в зашифрованном виде (Argon2id).

## Вход

1. Отправьте POST-запрос на `/auth/login`:
   ```json
   {
     "email": "user@example.com",
     "password": "ваш_пароль"
   }
   ```
2. Ответ аналогичен регистрации — пара токенов.

## Обновление токена

Когда access_token истекает, используйте refresh_token:

1. POST `/auth/refresh`:
   ```json
   { "refresh_token": "ваш_refresh_token" }
   ```
2. Получите новую пару токенов. Старый refresh_token становится недействительным.

> Если refresh_token уже был использован ранее — все активные сессии аккаунта будут немедленно аннулированы (защита от кражи токенов).

## Выход

- **Выход из текущей сессии:** POST `/auth/logout` с телом `{ "refresh_token": "..." }`
- **Выход из всех сессий:** POST `/auth/logout-all` (требует только access_token в заголовке)
