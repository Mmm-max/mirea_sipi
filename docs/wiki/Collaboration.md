# Совместное планирование

## Создание встречи

1. POST `/meetings`:
   ```json
   {
     "title": "Планирование спринта",
     "description": "Ежеквартальное планирование",
     "duration_minutes": 60,
     "priority": "high",
     "search_range_start": "2025-06-10T00:00:00Z",
     "search_range_end": "2025-06-14T23:59:00Z",
     "earliest_start_time": "09:00",
     "latest_start_time": "18:00"
   }
   ```
2. Организатор автоматически добавляется как принятый участник.
3. Встреча создаётся со статусом `draft` до выбора времени.

## Добавление участников

POST `/meetings/:id/participants`:
```json
{ "participant_ids": ["uuid-1", "uuid-2"] }
```
Участники получают уведомление и статус приглашения `pending`.

## Поиск свободных слотов

POST `/meetings/:id/search-slots`:
```json
{ "top_n": 5 }
```
Система проверяет для каждого кандидатного окна (шаг 15 минут):
- рабочие часы каждого участника
- периоды недоступности (отпуска, командировки)
- существующие события в календаре каждого участника
- доступность забронированных ресурсов (переговорных комнат и т.д.)

Каждому слоту присваивается оценка от 0 до 100. Слоты с оценкой 0 имеют жёсткие конфликты.

**Пример ответа:**
```json
{
  "slots": [
    {
      "rank": 1,
      "score": 85,
      "start_at": "2025-06-11T09:00:00Z",
      "end_at": "2025-06-11T10:00:00Z",
      "conflicts": [
        { "conflict_type": "event_conflict", "visible_title": "Ревью кода", "visible_priority": "low" }
      ]
    }
  ]
}
```

## Выбор слота

POST `/meetings/:id/select-slot`:
```json
{ "meeting_slot_id": "uuid-слота" }
```
- Нельзя выбрать слот с жёсткими конфликтами.
- После выбора статус встречи меняется на `scheduled`.
- Участники получают уведомление о подтверждённом времени.

## Ответ на приглашение

POST `/meetings/:id/respond`:
```json
{ "response_status": "accepted" }
```
Доступные статусы: `accepted`, `declined`, `tentative`, `pending`.

## Предложение альтернативного времени

Если предложенное время не подходит:

POST `/meetings/:id/request-alternative`:
```json
{
  "proposed_start_at": "2025-06-12T14:00:00Z",
  "proposed_end_at": "2025-06-12T15:00:00Z",
  "comment": "Первая половина дня занята"
}
```
Статус участника становится `tentative`, организатор получает уведомление.

## Уведомления

GET `/notifications` — список всех ваших уведомлений.

Уведомления создаются автоматически при:
- получении приглашения на встречу
- изменении параметров встречи
- выборе времени встречи
- предложении альтернативного времени участником
- конфликте ресурса

POST `/notifications/:id/read` — отметить одно уведомление как прочитанное.
POST `/notifications/read-all` — отметить все как прочитанные.
